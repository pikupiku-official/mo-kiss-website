# モーキス 公式サイト

1999年風のレトロなギャルゲー公式サイト with PHP + MySQL CMS

## 概要

「モーキス」は、1999年7月を舞台にした恋愛シミュレーションゲームの公式サイトです。
このプロジェクトは、1999年当時のインターネット文化を再現しつつ、
裏側には現代的なCMS機能を備えたWebサイトです。

### 特徴

- **1999年ど真ん中のレトロデザイン**
  - フレームセット構造
  - テーブルレイアウト
  - 原色系の色使い
  - アニメーションGIF
  - 工事中ページ

- **現代的な管理機能**
  - PHPベースのCMS
  - MySQLデータベース
  - 認証付き管理画面
  - コンテンツの動的管理

- **1999年風演出**
  - アクセスカウンター
  - BGM自動再生（ON/OFF切替可）
  - カウントダウン表示
  - レトロなアイコン

## 動作環境

### サーバー要件

- PHP 7.4以上
- MySQL 5.7以上 または MariaDB 10.2以上
- Apache または Nginx
- GDライブラリ（カウンター画像生成用）

### 推奨環境

- PHP 8.0以上
- MySQL 8.0以上

## セットアップ手順

### 1. ファイルのアップロード

プロジェクトファイル一式をサーバーのドキュメントルートにアップロードします。

```bash
# FTPまたはSCPでアップロード
scp -r * user@server:/path/to/webroot/
```

### 2. データベースの作成

MySQLにログインし、データベースを作成します。

```sql
CREATE DATABASE mokiss_site DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. データベースの初期化

`database.sql` を実行してテーブルと初期データを作成します。

```bash
mysql -u ユーザー名 -p mokiss_site < database.sql
```

または、phpMyAdminなどのツールでSQLファイルをインポートします。

### 4. 設定ファイルの編集

`includes/config.php` を開き、データベース接続情報を設定します。

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'mokiss_site');
define('DB_USER', 'your_username');  // 変更してください
define('DB_PASS', 'your_password');  // 変更してください
```

**重要**: 本番環境では必ず以下も変更してください：

```php
// エラー表示設定（本番環境では必ず無効化）
ini_set('display_errors', 0);
error_reporting(0);
```

### 5. ディレクトリのパーミッション設定

以下のディレクトリに書き込み権限を設定します。

```bash
chmod 755 sounds/bgm
chmod 755 images/characters
chmod 755 images/icons
```

### 6. 動作確認

ブラウザで以下のURLにアクセスして動作を確認します。

- 公開サイト: `http://yourdomain.com/`
- 管理画面: `http://yourdomain.com/admin/`

### 7. 初回ログイン

管理画面に以下のデフォルトアカウントでログインします。

- **ユーザー名**: `admin`
- **パスワード**: `admin123`

**セキュリティのため、初回ログイン後すぐにパスワードを変更してください！**

## 使い方

### 管理画面の各機能

#### 1. What's New管理

サイトの更新情報を追加・編集・削除できます。
新しい情報はトップページに自動的に表示されます。

#### 2. STORY管理

ゲームのストーリー紹介ページを編集できます。
HTMLタグが使用できるため、自由にレイアウトできます。

#### 3. キャラクター管理

6人のヒロインの情報を管理します。
- 名前、ふりがな
- 学年、部活、身長
- 紹介文
- 画像パス

画像は手動で `images/characters/` にアップロードしてください。

#### 4. SYSTEM管理

ゲームシステムの説明ページを編集できます。

#### 5. SPECIAL管理

発売情報と発売日カウントダウンを管理します。
発売予定日を設定すると、自動的にカウントダウンが表示されます。

#### 6. BGM管理

サイトBGMの設定を管理します。
- BGMの有効/無効切替
- BGMファイルのパス指定

BGMファイル（MP3推奨）は手動で `sounds/bgm/` にアップロードしてください。

#### 7. ユーザー管理

管理画面にログインできるユーザーを管理します。
複数の管理者を追加できます。

### 画像のアップロード

画像は以下のディレクトリに配置します：

- キャラクター画像: `images/characters/`
- ロゴ画像: `images/logo/`
- アイコン: `images/icons/`
- 背景: `images/backgrounds/`

詳細は [images/IMAGE_GUIDE.md](images/IMAGE_GUIDE.md) を参照してください。

### BGMのアップロード

1. MP3形式の音楽ファイルを準備
2. `sounds/bgm/` にアップロード
3. 管理画面の「BGM管理」でパスを設定

## セキュリティに関する注意

### 必須対応項目

1. **デフォルトパスワードの変更**
   - 初回ログイン後すぐに変更してください

2. **データベース認証情報の保護**
   - `includes/config.php` のパーミッションを適切に設定

3. **エラー表示の無効化**
   - 本番環境では `config.php` の `display_errors` を0に設定

4. **HTTPSの使用**
   - 可能な限りHTTPSで運用してください

5. **定期的なバックアップ**
   - データベースとファイルを定期的にバックアップ

### 実装済みセキュリティ機能

- SQLインジェクション対策（プリペアドステートメント）
- XSS対策（htmlspecialchars）
- パスワードハッシュ化（password_hash）
- セッション管理
- セッションタイムアウト（1時間）

## トラブルシューティング

### データベース接続エラー

`includes/config.php` の設定を確認してください。

### 画像が表示されない

1. パーミッションを確認
2. 管理画面で指定したパスが正しいか確認
3. ファイルが実際にアップロードされているか確認

### BGMが再生されない

1. ファイル形式がMP3か確認
2. ブラウザの自動再生ポリシーによりブロックされている可能性
   - ユーザーが「ON」ボタンを押す必要があります

### フレームが表示されない

一部の現代的なブラウザやセキュリティ設定でフレームが無効化されている場合があります。
この場合は各ページに直接アクセスしてください。

## レトロ演出について

### フレームセット

HTML4.01のframesetタグを使用しています。
現代のHTML5では非推奨ですが、1999年の再現のため意図的に使用しています。

### MIDI BGM

1999年当時はMIDI音源が主流でしたが、現代のブラウザでは再生できないため、
MP3ファイルを使用しつつ「MIDI」と表記することで雰囲気を出しています。

### アクセスカウンター

訪問者数を数字画像で表示します。
画像は `api/counter.php` で動的に生成されます。
