# モーキス公式サイト セットアップガイド

このドキュメントでは、お名前.com RSプランでのセットアップ手順を詳しく説明します。

## 事前準備

### 必要なもの

- お名前.com RSプランのサーバー契約
- FTPクライアント（FileZilla、WinSCP等）
- テキストエディタ
- このプロジェクトのファイル一式

### 確認事項

お名前.com RSプランの管理画面で以下を確認してください：

- PHPバージョン: 7.4以上（推奨: 8.0以上）
- MySQLが利用可能
- FTP接続情報（ホスト、ユーザー名、パスワード）

## ステップ1: データベースの作成

### 1-1. コントロールパネルにログイン

お名前.comのコントロールパネルにログインします。

### 1-2. データベース作成

1. 「データベース」メニューを選択
2. 「MySQL」を選択
3. 「データベース追加」をクリック

**設定例**:
- データベース名: `mokiss_site`
- 文字コード: `UTF-8 (utf8mb4)`

### 1-3. データベースユーザーの作成

1. 「ユーザー追加」をクリック
2. ユーザー名とパスワードを設定
3. データベースに対する権限を「すべて」に設定

**メモしておく情報**:
- データベース名
- ユーザー名
- パスワード
- ホスト名（通常は `localhost`）

## ステップ2: ファイルのアップロード

### 2-1. FTP接続

FTPクライアントで接続します。

**接続情報**:
- ホスト: お名前.comから提供された情報
- ユーザー名: FTPユーザー名
- パスワード: FTPパスワード
- ポート: 21 (通常)

### 2-2. ファイルのアップロード

1. サーバーの `public_html` または `www` ディレクトリに移動
2. プロジェクトのすべてのファイルをアップロード

**アップロードするファイル**:
```
index.html
main.html
menu.html
home.php
story.php
character.php
system.php
special.php
construction.html
database.sql
css/
images/
sounds/
includes/
api/
admin/
```

**所要時間**: 約5〜10分

## ステップ3: 設定ファイルの編集

### 3-1. config.phpの編集

FTPクライアントで `includes/config.php` を開き、編集します。

**編集箇所**:

```php
// データベース接続情報
define('DB_HOST', 'localhost');           // 通常はlocalhost
define('DB_NAME', 'mokiss_site');         // 作成したDB名
define('DB_USER', 'your_username');       // DBユーザー名
define('DB_PASS', 'your_password');       // DBパスワード

// サイトURL（本番環境）
define('SITE_URL', 'https://yourdomain.com');  // 実際のドメイン
```

### 3-2. エラー表示の無効化（本番環境）

```php
// エラー表示設定（本番環境では必ず無効化）
ini_set('display_errors', 0);
error_reporting(0);
```

**重要**: 必ず `display_errors` を `0` に変更してください！

### 3-3. 保存してアップロード

編集した `config.php` を保存し、サーバーにアップロードします。

## ステップ4: データベースの初期化

### 4-1. phpMyAdminにアクセス

お名前.comのコントロールパネルから phpMyAdmin を開きます。

### 4-2. SQLファイルのインポート

1. 左側から `mokiss_site` データベースを選択
2. 上部の「インポート」タブをクリック
3. 「ファイルを選択」から `database.sql` を選択
4. 「実行」ボタンをクリック

### 4-3. インポートの確認

以下のテーブルが作成されていることを確認：
- users
- news
- pages
- characters
- special
- settings
- access_counter

## ステップ5: パーミッションの設定

FTPクライアントで以下のディレクトリのパーミッションを設定します。

### 設定方法（FileZillaの場合）

1. ディレクトリを右クリック
2. 「ファイルの属性」を選択
3. 数値を入力

### 必要なパーミッション

```
sounds/bgm/           → 755
images/characters/    → 755
images/icons/         → 755
images/backgrounds/   → 755
images/logo/          → 755
```

**755の意味**:
- オーナー: 読み・書き・実行
- グループ: 読み・実行
- その他: 読み・実行

## よくある問題と解決方法

### Q1. 「データベース接続エラー」が表示される

**原因**:
- `config.php` の設定が間違っている
- データベースが作成されていない

**解決方法**:
1. `config.php` の接続情報を再確認
2. phpMyAdminでデータベースが存在するか確認

### Q2. 管理画面にログインできない

**原因**:
- `database.sql` がインポートされていない
- パスワードが間違っている

**解決方法**:
1. phpMyAdminで `users` テーブルが存在するか確認
2. デフォルトアカウント（admin/admin123）でログイン

### Q3. 画像が表示されない

**原因**:
- 画像がアップロードされていない
- パスが間違っている
- パーミッションが不足

**解決方法**:
1. FTPで画像ファイルが存在するか確認
2. 管理画面のパス設定を確認
3. ディレクトリのパーミッションを755に設定

### Q4. BGMが再生されない

**原因**:
- ファイルがアップロードされていない
- ブラウザの自動再生がブロックされている

**解決方法**:
1. BGMファイルが存在するか確認
2. ブラウザで「BGM ON」ボタンをクリック

### Q5. 「500 Internal Server Error」が表示される

**原因**:
- PHPのバージョンが古い
- ファイルのパーミッションが不適切
- 構文エラー

**解決方法**:
1. PHPバージョンを7.4以上に設定
2. パーミッションを確認
3. エラーログを確認
